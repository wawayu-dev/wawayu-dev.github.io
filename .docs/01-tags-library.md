## 📝 标签使用原则

### 标签组成（3-5个）

1. **主题标签**（必选1个）
   - 文章的核心主题
   - 例如：Spring、MySQL、Redis、Java

2. **技术标签**（必选1-2个）
   - 涉及的具体技术点
   - 例如：并发编程、分布式锁、索引优化

3. **场景/深度标签**（必选1-2个）
   - 应用场景：高并发、性能优化、问题排查
   - 文章深度：源码分析、最佳实践

## ⚠️ 注意事项

### 避免的做法

❌ **过于细分的标签**
```yaml
# 不好
tags: ["Spring AOP 动态代理", "JDK动态代理", "CGLIB代理"]

# 好
tags: ["Spring", "设计模式", "源码分析"]
```

❌ **同义词标签**
```yaml
# 不好
tags: ["并发", "并发编程", "多线程编程"]

# 好
tags: ["并发编程", "多线程"]
```

❌ **无意义标签**
```yaml
# 不好
tags: ["实战", "教程", "入门", "进阶"]

# 好
tags: ["最佳实践", "源码分析"]
```

### 推荐的做法

✅ **标签组合合理**
- 1个主题 + 1-2个技术 + 1-2个场景/深度

✅ **标签具有检索价值**
- 用户能通过标签快速找到相关文章

✅ **标签保持一致性**
- 同类文章使用相同的标签体系

## 🔄 标签维护

### 新增标签流程
1. 检查是否已有类似标签
2. 确认标签的必要性（是否有多篇文章会使用）
3. 在本文档中添加新标签
4. 更新 `.docs/04-tags-usage-statistics.md`

### 定期审查
- 每月检查标签使用情况
- 合并低频标签（使用次数 < 2）
- 删除无用标签
- 保持标签总数在 50 个以内
