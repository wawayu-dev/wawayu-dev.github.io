## 🎯 Front Matter 说明

### 必填字段

```yaml
title: "文章标题"
# 规则：简洁明了，突出核心主题
# 示例："Spring AOP 原理与动态代理实现深度解析"

date: 2024-12-15
# 规则：使用 YYYY-MM-DD 格式
# 建议：按实际创建日期填写

tags: ["标签1", "标签2", "标签3"]
# 规则：3-5个标签，必须从标签库中选择
# 组成：1个主题 + 1-2个技术 + 1-2个场景/深度
# 参考：.docs/01-tags-library.md

categories: ["技术"]
# 规则：固定使用 ["技术"]
# 说明：暂不使用其他分类

description: "文章简介"
# 规则：50-100字，一句话概括
# 作用：显示在文章列表和搜索结果中

draft: false
# 规则：false 表示发布，true 表示草稿
# 说明：草稿不会出现在生产环境
```

## 📐 文章结构规范

### 标题层级

```markdown
# 一级标题（不使用，由 Front Matter 的 title 生成）

## 二级标题（章节）

### 三级标题（小节）

#### 四级标题（细节，尽量少用）
```

### 开场白结构

```markdown
背景介绍（1-2段）
↓
本文内容（1段）
↓
本文亮点（3-5个）
```

### 正文结构

```markdown
## 核心概念
- 理论讲解
- 原理分析

## 实战应用
- 场景描述
- 代码示例
- 问题解决

## 最佳实践
- 实践建议
- 注意事项

## 总结与思考
- 核心要点
- 思考展望
```

## 💻 代码示例规范

### 代码块格式

\```java
// 1. 添加必要的注释
// 2. 确保代码可运行
// 3. 突出关键代码行

public class Example {
    /**
     * 方法说明
     */
    public void method() {
        // 实现逻辑
    }
}
\```

### 代码示例原则

1. **完整性**：提供完整可运行的代码
2. **简洁性**：只保留核心逻辑，去除无关代码
3. **注释性**：关键代码添加注释说明
4. **规范性**：遵循代码规范（命名、格式）
